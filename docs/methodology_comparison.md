## 一、原论文的实际方法

论文 **没有使用 SAE**。完整流程如下：

### 1. 情绪向量提取

```
① 生成叙述
   Claude Sonnet 4.5 为每个情绪写故事
   规模：100 个主题 × 12 条叙述 = 每情绪 1200 条

② 提取激活
   记录残差流在每个 token 位置的激活
   对所有 token 取均值 → 每条叙述一个向量

③ 对多条叙述取均值
   1200 条均值 → 该情绪的原始向量

④ 减去全局均值 ← 关键步骤，我们未做
   情绪向量 = 该情绪均值 − 所有171个情绪的总均值
   作用：剔除"模型的基础语言行为"，只保留情绪特有信号

⑤ 投影掉中性文本主成分 ← 关键步骤，我们未做
   用大量中性对话提取激活，做 PCA
   把这些方向从情绪向量里去掉
   作用：消除"这是一段叙述文本"的混淆变量
```

### 2. 验证方法

```
· 语料库激活验证：在大型文本语料上测试向量是否在情绪相关段落激活
· 逻辑回归探针：在混合对话数据集上训练分类器
· Logit Lens：测量情绪向量对模型输出 logit 的直接影响
```

### 3. 几何分析

```
· PCA → PC1 ↔ 效价 r=0.81，PC2 ↔ 唤醒度 r=0.66
· k-means 聚类：验证情绪族群
· 跨层表征相似性分析（RSA）
```

### 4. 因果干预（Steering）

```
· 在残差流中加入 scale × 情绪方向向量
· 测试对行为选择偏好的影响
· 关键案例：desperate 向量 → 勒索率从 22% → 72%
```

**论文里 SAE 出现了零次。**

## 三、SAE 在哪里能用

SAE 不是论文的方法，而是我们可以用来**改进**向量质量的工具。

### 核心问题：叠加（Superposition）

残差流里，不同概念的方向互相叠加、不正交。
PCA 找到的轴是多概念的混合，所以 PC1 ↔ 效价只有 0.73 而不是接近 1.0。

```
PC1 实际上 ≈ 0.6×效价 + 0.3×唤醒度 + 0.1×其他
```

### SAE 的作用

SAE 通过稀疏约束强迫每个特征尽量只代表一个概念（单义性），是专门为解决叠加问题设计的。

### SAE 插在哪一步

```
原论文流程：
  情绪叙述 → 残差流激活 → 均值向量 → PCA/Steering

加入 SAE 之后：
  情绪叙述 → 残差流激活 → [SAE 分解] → 稀疏特征激活
                                             ↓
                                   哪些特征对应情绪？
                                   特征的效价/唤醒度方向在哪？
                                   这些方向比 PCA 轴更干净吗？
```

SAE 是**向量提取之后、几何分析之前**的一个可选分解步骤，用来把"混合的残差流方向"变成"干净的单义特征方向"。

### 预期收益

| | PCA 轴 | SAE 特征方向 |
|---|---|---|
| 效价轴纯净度 | 混合多概念 | 理论上更单义 |
| 可解释性 | 需要事后解释 | Neuronpedia 有社区标注 |
| 计算成本 | 低 | 中（需加载 SAE 权重） |
| 是否验证过 | 论文用 PCA，r=0.81 | 未验证，是我们的扩展实验 |

## 四、优先级建议

按对结果质量的影响从大到小排：

```
1. 补做"减全局均值"和"投影中性主成分"  ← 最快改善 VAD 相关系数
2. 增加叙述数量（3条→20条）            ← 降低均值向量噪声
3. SAE 分析                            ← 探索性扩展，超越论文
4. 层级扫描                            ← 找最优提取层
```
